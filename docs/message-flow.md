# メッセージ処理の流れ

ユーザーが Slack にメッセージを送ってから、AI 秘書が返信するまでの処理の流れを解説します。

## 基本フロー

```
ユーザー                Slack               SlackBot            ClaudeExecutor        Claude CLI
  │                     │                    │                    │                    │
  │  メッセージ送信       │                    │                    │                    │
  │────────────────────>│                    │                    │                    │
  │                     │  Socket Mode       │                    │                    │
  │                     │  (WebSocket)       │                    │                    │
  │                     │───────────────────>│                    │                    │
  │                     │                    │                    │                    │
  │                     │                    │ ① イベント検証       │                    │
  │                     │                    │  ・重複チェック       │                    │
  │                     │                    │  ・チャンネル確認     │                    │
  │                     │                    │  ・Bot除外           │                    │
  │                     │                    │                    │                    │
  │                     │  リアクション付与    │                    │                    │
  │  👀 (処理中)         │<───────────────────│                    │                    │
  │<────────────────────│                    │                    │                    │

  │                     │                    │                    │                    │
  │                     │                    │ ③ Claude 呼び出し   │                    │

  │                     │                    │                    │───────────────────>│
  │                     │                    │                    │                    │

  │                     │                    │                    │                    │
  │                     │                    │                    │  JSON レスポンス     │
  │                     │                    │                    │<───────────────────│
```

## 各ステップの詳細

### ① イベント検証（handle_event）

Slack から受信したイベントに対して、以下の順にフィルタリングを行います。
すべてを通過したメッセージだけが処理対象になります。

```
イベント受信
  │
  ├─ type が "events_api" でない → 無視
  │
  ├─ シャットダウン中 → 無視
  │
  ├─ event_id が処理済み → 無視（重複排除）
  │
  ├─ event.type が "message" でない → 無視
  │
  ├─ bot_id がある（Bot のメッセージ）→ 無視
  │
  ├─ subtype がある（編集・削除など）→ 無視
  │   └─ ただし "file_share" は通過（ファイル添付）
  │
  ├─ チャンネルが未登録 → 無視
  │
  └─ ✅ 新しいスレッドで process_message を実行
```

### ② セッション管理

Slack のスレッド（`thread_ts`）と Claude Code のセッション（`session_id`）を SQLite で紐づけます。

```
1回目のメッセージ（新規スレッド）:
  SessionStore.find → nil（セッションなし）
  Claude CLI を session_id なしで実行
  → 新しい session_id が返ってくる
  SessionStore.save で保存

2回目以降のメッセージ（既存スレッド）:
  SessionStore.find → "abc-123"（既存セッション）
  Claude CLI を --resume abc-123 で実行
  → 前回の会話の続きとして応答
  SessionStore.save で更新
```

これにより、同じスレッド内のメッセージは **会話の文脈を維持** したまま Claude と対話できます。

### ③ Claude CLI 呼び出し

ClaudeExecutor が組み立てるコマンド:

```bash
claude -p "ユーザーのメッセージ" \
  --output-format json \
  --dangerously-skip-permissions \
  --append-system-prompt "System Promptの内容" \  # チャンネルに設定がある場合
  --resume "session_id"                           # 既存セッションがある場合
```

- サブプロセスとして実行（`Open3.popen3`）
- 30 分のタイムアウト付き
- `working_directory` が設定されていれば、そのディレクトリで実行（プロジェクト固有の操作が可能に）

### ④ セッション保存

Claude CLI から返された `session_id` を SQLite に保存します。次回同じスレッドにメッセージが来たとき、この `session_id` を使って会話を継続します。

### ⑤ 出力変換

Claude の出力は Markdown 形式ですが、Slack は独自の mrkdwn 記法を使うため、SlackMrkdwnConverter で変換します。

```
Claude の出力:                    Slack に投稿:
### タスク一覧                →   *タスク一覧*
**重要** な更新です            →   *重要* な更新です
[リンク](https://example.com) →   <https://example.com|リンク>
```

## ファイル添付の処理

ユーザーが画像や PDF を添付した場合の追加フロー:

```
ユーザー                SlackBot
  │                      │
  │  画像 + テキスト       │
  │─────────────────────>│
  │                      │
  │                      │ ファイルの MIME タイプを確認
  │                      │  ├ JPEG, PNG, GIF, WebP, PDF → サポート対象
  │                      │  └ それ以外 → 非サポート
  │                      │
  │                      │ サポート対象ファイルをダウンロード
  │                      │  ├ Slack API でファイル取得
  │                      │  └ 一時ディレクトリに保存
  │                      │
  │                      │ クエリを組み立て
  │                      │  "ユーザーのテキスト
  │                      │
  │                      │   添付ファイル: /tmp/slack_files_xxx/image.png"
  │                      │
  │                      │ Claude CLI に渡す（Claude がファイルを読み取る）
  │                      │
  │                      │ 処理完了後、一時ディレクトリを削除
```

## 起動時の処理

```
bin/start 実行
  │
  ├─ 環境変数の読み込み（.env ファイル）
  │
  ├─ 全コンポーネントの初期化
  │   ├─ SQLite テーブルの自動作成
  │   └─ チャンネルルーティングの構築
  │
  ├─ 孤立リアクションのクリーンアップ（ActiveReactionStore）
  │
  └─ Socket Mode でイベントループ開始
```
